## 3. 复现中遇到的主要困难

### 3.1 Windows 原生扩展编译不稳定

主要问题：

- `torch_scatter` 在 Windows + MSVC 下编译参数不兼容
- `nvdiffrast` 如果走全局安装，容易碰权限和环境污染
- `slangtorch` / `BVH` 这条链对工具链路径很敏感

处理：

- 本地 vendor
- 强制 `HostX64\\x64`
- 修掉 `-O3`
- 优先使用仓库内构建结果


### 3.2 仓库路径含空格导致 `slangtorch` 出错

当前仓库路径是：

`D:\Grade Three Down\Experiment\SVG-IR`

`slangtorch` 在 Windows 下解析 `.depfile` 时，会把带空格路径截断成类似 `D:\Grade\...`，导致 `stage2` 和评测链直接报错。

处理：

- 训练和评测实际运行时改成通过无空格路径别名来执行
- 这属于纯工程修复，不改算法逻辑


### 3.3 `eval_nvs.py` 对旧 `cfg_args` 不兼容

问题：

- 脚本直接访问 `args.max_views`
- 但训练目录里的 `cfg_args` 可能没有这个字段

处理：

- 改为 `getattr(args, "max_views", None)`


### 3.4 `eval_relighting_tensoIR.py` 依赖外部 HDR 资产

问题：

- 仓库并没有自带完整 `env_map/Environment_Maps/high_res_envmaps_1k/...`
- 官方实际上也是要求自己准备 HDR 再改脚本路径

处理：

- 本地补了 `fireplace / forest / night` 三张 HDR
- 并把脚本路径改成仓库内 `env_map/*.hdr`


### 3.5 `imageio` 想下载 `freeimage` 到用户目录

问题：

- [scene/envmap.py](D:\Grade Three Down\Experiment\SVG-IR\scene\envmap.py) 原本调用 `imageio.plugins.freeimage.download()`
- 默认写到 `C:\Users\...\AppData\Local\imageio`
- 当前环境对那个目录没有写权限

处理：

- 移除这条依赖
- `.hdr` 改为直接使用已有的 OpenCV 读取函数 `read_hdr()`


## 4. 哪些地方和官方一样，哪些地方不一样

### 4.1 一样的地方

当前流程与官方保持一致的核心部分：

- 两阶段训练顺序不变
- `stage1 -> stage2` checkpoint 传递方式不变
- `stage1` 迭代数：`30000`
- `stage2` 迭代数：`50000`
- `stage2 sample_num = 64`
- `stage2 env_resolution = 32`
- 使用 `train.py -> eval_nvs.py -> eval_relighting_tensoIR.py` 这条主流程


### 4.2 不一样的地方

不一致点主要是本地可运行性相关：

1. **分辨率**
   - 官方默认是 `resolution = -1`
   - 本次本地复现使用了 `-r 4`
   - 这是最主要的不一致项

2. **HDR 资产**
   - 官方评测依赖外部 HDR
   - 这次我补的是本地可用的公开 HDR，并不保证和作者原始使用的 HDR 完全同源

3. **若干工程补丁**
   - 本地原生模块路径注入
   - `eval_nvs.py` 的兼容修复
   - `scene/envmap.py` 的 HDR 读取修复
   - Windows 路径空格规避

4. **训练脚本行为**
   - 为了稳定本地训练，我保留了 `--skip_final_eval`
   - 也就是说训练和评测是分开执行的，而不是训练结束后自动全评测


### 4.3 这意味着什么

当前结果可以称为：

`官方流程骨架一致 + 本地分辨率降档 + 工程兼容修复`

不能称为：

`论文原配置逐项严格对齐复现`

如果你后面要往“更接近论文结果”推进，优先级应该是：

1. 把 `-r 4` 提升到 `-r 2` 或默认分辨率
2. 尽量找到与作者一致的 HDR 资产
3. 再比较数值

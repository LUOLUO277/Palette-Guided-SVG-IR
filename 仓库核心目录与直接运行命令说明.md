## 4. “先训练 30000 轮，再训练 20000 轮”到底是哪来的

你看到的原始脚本是：

```bash
python train.py --eval \
    -s ${root_dir}${i} \
    -m output/TensoIR/${i}/gss \
    ...

python train.py --eval \
    -s ${root_dir}${i} \
    -m output/TensoIR/${i}/render_relight \
    -c output/TensoIR/${i}/gss/chkpnt30000.pth \
    --iterations 50000 \
    ...
```

你会疑惑的点是：

- 原脚本明明没有写“再训练 20000 轮”
- 为什么最后大家都说 Stage 2 是“从 30000 续训到 50000”

答案在 [train.py](D:\Grade Three Down\Experiment\SVG-IR\train.py) 里。

### 4.1 `checkpoint` 会恢复当前迭代数

在 [train.py:38](D:\Grade Three Down\Experiment\SVG-IR\train.py:38) 到 [train.py:40](D:\Grade Three Down\Experiment\SVG-IR\train.py:40)：

```python
if args.checkpoint:
    print("Create Gaussians from checkpoint {}".format(args.checkpoint))
    first_iter = gaussians.create_from_ckpt(args.checkpoint, restore_optimizer=True)
```

也就是说：

- 如果你传了 `-c chkpnt30000.pth`
- `first_iter` 就会被恢复成 `30000`

### 4.2 训练循环不是从 1 跑到 50000，而是从 `first_iter + 1` 跑到 `iterations`

在 [train.py:109](D:\Grade Three Down\Experiment\SVG-IR\train.py:109) 到 [train.py:110](D:\Grade Three Down\Experiment\SVG-IR\train.py:110)：

```python
progress_bar = tqdm(range(first_iter + 1, opt.iterations + 1), ...)
```

这句是关键。

如果：

- `first_iter = 30000`
- `opt.iterations = 50000`

那么实际循环区间就是：

```text
30001, 30002, ..., 50000
```

也就是：

```text
50000 - 30000 = 20000 轮
```

所以：

- Stage 1：从 1 跑到 30000
- Stage 2：从 30001 跑到 50000

这就是“先训练 30000 轮，再训练 20000 轮”的来源。

### 4.3 这不是你本地脚本自己发明的，而是官方原始命令本来就隐含这个逻辑

核心不是 `ps1` 发明了“20000”。

真正来源是：

1. 官方第二阶段命令传了 `-c chkpnt30000.pth`
2. 同时又传了 `--iterations 50000`
3. `train.py` 会从 checkpoint 恢复 `first_iter`
4. 训练循环从 `first_iter + 1` 开始

所以，“30000 + 20000”其实是官方原始训练逻辑本来就隐含的结果。

## 5. 当前本地方法和官方原始脚本的关系

### 5.1 完全一致的地方

主干训练流程是一致的：

1. 先 `gss`
2. 再 `render_relight`
3. 第二阶段从第一阶段 `chkpnt30000.pth` 接着训
4. 第二阶段目标迭代数是 `50000`

### 5.2 当前本地和官方原始脚本不同的地方

当前本地主要改了这些运行参数：

- Stage 1 输出目录名  
  官方：`output/TensoIR/${i}/gss`  
  本地：`output/TensoIR/armadillo/gss_local_r4_i30000`

- Stage 2 输出目录名  
  官方：`output/TensoIR/${i}/render_relight`  
  本地：`output/TensoIR/armadillo/render_relight_local_r4_i50000`

- 本地显式加了 `-r 4`
- 本地显式加了 `--skip_final_eval`
- 本地把保存间隔和 checkpoint 间隔固定成了终点迭代数
- 本地把 `save_training_vis_iteration` 改成了 `1000`

所以更准确的说法是：

- `训练阶段切分方式` 来自官方原始逻辑
- `本地命名方式和部分运行参数` 是你当前机器上的本地化版本

## 6. 最短结论

如果你只想记住最重要的 3 句话，那就是：

1. 当前真正的运行入口是 `train.py`、`eval_nvs.py`、`eval_relighting_tensoIR.py`。
2. 当前本地实际执行时，是先把仓库映射成 `X:\`，然后运行 `X:\train.py ...` 这类命令。
3. “先训 30000，再训 20000”不是本地脚本拍脑袋定的，而是官方原始命令配合 `train.py` 的 checkpoint 恢复逻辑自然得到的。
